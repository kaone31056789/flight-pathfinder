================================================================================
                              1. ABSTRACT
================================================================================

The Flight Route Pathfinder is an advanced web-based application that 
implements multiple graph traversal and shortest path algorithms to find 
optimal flight routes between Indian airports. The system demonstrates 
practical applications of data structures like graphs, heaps, and queues 
in real-world scenarios. It provides an interactive visualization of flight 
routes with realistic curved great-circle paths on an interactive map.

The project features a high-performance C++ algorithm core integrated with 
a Python Flask web server for optimal performance. The C++ backend implements 
four different pathfinding algorithms: Dijkstra's Algorithm, A* Search, 
Breadth-First Search (BFS), and Depth-First Search (DFS). The system offers 
both distance and time-based optimization metrics with an aesthetically 
designed user interface featuring advanced animations.

================================================================================
                            2. INTRODUCTION
================================================================================

Air travel is one of the most complex transportation networks, involving 
thousands of airports connected through millions of flight routes. Finding 
the optimal path between two locations is a classic graph theory problem 
that has significant real-world applications.

This project simulates a flight booking system's route optimization module, 
where users can find the best route between airports based on different 
criteria (shortest distance or minimum flight time). The application 
demonstrates how different graph algorithms perform on the same dataset 
and provides comparative insights.

The system uses 12 major Indian airports as nodes and their flight 
connections as weighted edges, creating a realistic flight network graph. 
All distances are calculated using the Haversine formula implemented in C++ 
to ensure geographical accuracy and optimal performance.

                        7. DATA STRUCTURES USED
================================================================================

1. GRAPH (ADJACENCY LIST)
   Type: Weighted Directed Graph
   Representation: Dictionary/HashMap
   Structure: { airport_code: [(neighbor, distance, time), ...] }
   
   Example:
   "DEL": [("BOM", 1148, 140), ("BLR", 1742, 195), ...]
   
   Advantages:
   - O(1) neighbor access
   - Memory efficient for sparse graphs
   - Easy to add/remove edges
   
   Space Complexity: O(V + E) where V = airports, E = flights

2. PRIORITY QUEUE (MIN-HEAP)
   Used in: Dijkstra's Algorithm, A* Search
   Implementation: C++ STL priority_queue<pair<int, string>>
   
   Operations:
   - Insert: O(log n)
   - Extract-Min: O(log n)
   - Decrease-Key: O(log n)
   
   Purpose: Efficiently select minimum distance node in C++ algorithms

3. QUEUE (FIFO)
   Used in: Breadth-First Search (BFS)
   Implementation: C++ STL queue<string>
   
   Operations:
   - Enqueue: O(1)
   - Dequeue: O(1)
   
   Purpose: Level-order traversal of graph in C++ BFS

4. STACK (LIFO)
   Used in: Depth-First Search (DFS)
   Implementation: C++ recursion stack / C++ STL stack<string>
   
   Operations:
   - Push: O(1)
   - Pop: O(1)
   
   Purpose: Backtracking exploration in C++ DFS

5. HASH MAP (UNORDERED_MAP)
   Used in: Parent tracking, distance tracking, visited set
   Implementation: C++ STL unordered_map<string, T>
   
   Operations:
   - Insert/Update: O(1) average
   - Lookup: O(1) average
   
   Purpose: Fast lookups and updates

6. ARRAY / LIST
   Used in: Path reconstruction, coordinate storage
   
   Purpose: Sequential data storage

ALGORITHM 1: DIJKSTRA'S ALGORITHM (C++ IMPLEMENTATION)
------------------------------------------------------
Type: Single-Source Shortest Path
Category: Greedy Algorithm
Optimality: Guaranteed optimal for non-negative weights
Implementation: C++ with STL priority_queue

C++ PSEUDOCODE:
```cpp
1. unordered_map<string, int> distances;
2. distances[source] = 0; // Initialize source
3. priority_queue<pair<int,string>, vector<>, greater<>> minHeap;
4. minHeap.push({0, source});
5. While (!minHeap.empty()):
   a. auto [currentDist, current] = minHeap.top();
   b. minHeap.pop();
   c. For each neighbor in graph[current]:
      - int newDist = distances[current] + edgeWeight;
      - If (newDist < distances[neighbor]):
        * distances[neighbor] = newDist;
        * parent[neighbor] = current;
        * minHeap.push({newDist, neighbor});
6. Reconstruct path using parent map
```

TIME COMPLEXITY: O((V + E) log V)
- V = number of airports (12)
- E = number of flights (57)
- Log V factor from C++ priority_queue operations

SPACE COMPLEXITY: O(V)
- Distance unordered_map: O(V)
- Parent unordered_map: O(V)
- Priority queue: O(V) worst case

C++ OPTIMIZATIONS:
- Use of STL priority_queue for efficient min-heap
- unordered_map for O(1) average lookups
- Pass by reference to avoid copying
- Reserve capacity for vectors

WHEN TO USE:
- Need guaranteed optimal path
- Graph has non-negative weights
- All edges should be considered

EXAMPLE OUTPUT:
DEL → BOM → BLR (Total: 1990 km)


ALGORITHM 2: A* SEARCH ALGORITHM (C++ IMPLEMENTATION)
-----------------------------------------------------
Type: Informed Search / Heuristic Search
Category: Best-First Search with Heuristic
Optimality: Guaranteed optimal with admissible heuristic
Implementation: C++ with custom heuristic function

HEURISTIC FUNCTION (C++):
h(n) = Haversine distance from current node to destination

C++ Haversine Formula Implementation:
```cpp
double haversine(double lat1, double lon1, double lat2, double lon2) {
    const double R = 6371.0; // Earth radius in km
    double phi1 = lat1 * M_PI / 180.0;
    double phi2 = lat2 * M_PI / 180.0;
    double dphi = (lat2 - lat1) * M_PI / 180.0;
    double dlambda = (lon2 - lon1) * M_PI / 180.0;
    
    double a = sin(dphi/2) * sin(dphi/2) + 
               cos(phi1) * cos(phi2) * sin(dlambda/2) * sin(dlambda/2);
    double c = 2 * atan2(sqrt(a), sqrt(1-a));
    return R * c;
}
```

C++ PSEUDOCODE:
```cpp
1. unordered_map<string, double> g_score, f_score;
2. g_score[source] = 0;
3. f_score[source] = haversine(source, destination);
4. priority_queue<pair<double,string>, vector<>, greater<>> openSet;
5. openSet.push({f_score[source], source});
6. While (!openSet.empty()):
   a. auto [_, current] = openSet.top(); openSet.pop();
   b. If (current == destination): return reconstructPath();
   c. For each neighbor in graph[current]:
      - double tentative_g = g_score[current] + edgeWeight;
      - If (tentative_g < g_score[neighbor]):
        * g_score[neighbor] = tentative_g;
        * f_score[neighbor] = tentative_g + haversine(neighbor, dest);
        * parent[neighbor] = current;
        * openSet.push({f_score[neighbor], neighbor});
```

TIME COMPLEXITY: O(E log V)
- Better than Dijkstra in practice due to heuristic guidance
- Explores fewer nodes

SPACE COMPLEXITY: O(V)

ADVANTAGES OVER DIJKSTRA:
- Faster path finding with good heuristic
- Explores nodes toward goal first
- Reduces search space

EXAMPLE OUTPUT:
BLR → HYD → DEL (Guided by straight-line distance)


ALGORITHM 3: BREADTH-FIRST SEARCH (BFS) - C++ IMPLEMENTATION
------------------------------------------------------------
Type: Unweighted Shortest Path / Level-Order Traversal
Category: Graph Traversal
Optimality: Optimal for unweighted graphs (minimum hops)
Implementation: C++ with STL queue

C++ PSEUDOCODE:
```cpp
1. queue<string> q;
2. unordered_set<string> visited;
3. unordered_map<string, string> parent;
4. q.push(source);
5. visited.insert(source);
6. While (!q.empty()):
   a. string current = q.front(); q.pop();
   b. If (current == destination): return reconstructPath();
   c. For (const auto& neighbor : graph[current]):
      - If (visited.find(neighbor) == visited.end()):
        * visited.insert(neighbor);
        * parent[neighbor] = current;
        * q.push(neighbor);
```

TIME COMPLEXITY: O(V + E)
- Each vertex visited once: O(V)
- Each edge explored once: O(E)

SPACE COMPLEXITY: O(V)
- Queue size: O(V)
- Visited set: O(V)

CHARACTERISTICS:
- Finds path with minimum number of hops
- Ignores edge weights (distance/time)
- Explores all neighbors before going deeper

EXAMPLE OUTPUT:
DEL → BLR (Direct, 1 hop) vs DEL → BOM → GOI → BLR (3 hops)


ALGORITHM 4: DEPTH-FIRST SEARCH (DFS) - C++ IMPLEMENTATION
----------------------------------------------------------
Type: Graph Traversal
Category: Backtracking Search
Optimality: Not guaranteed (finds "a" path, not optimal)
Implementation: C++ recursive approach

C++ ITERATIVE VERSION:
```cpp
1. stack<string> s;
2. unordered_set<string> visited;
3. unordered_map<string, string> parent;
4. s.push(source);
5. While (!s.empty()):
   a. string current = s.top(); s.pop();
   b. If (current == destination): return reconstructPath();
   c. If (visited.find(current) != visited.end()): continue;
   d. visited.insert(current);
   e. For (const auto& neighbor : graph[current]):
      - If (visited.find(neighbor) == visited.end()):
        * parent[neighbor] = current;
        * s.push(neighbor);
```

C++ RECURSIVE VERSION:
```cpp
bool dfs(const string& current, const string& destination,
         unordered_set<string>& visited,
         unordered_map<string, string>& parent) {
    if (current == destination) return true;
    
    visited.insert(current);
    
    for (const auto& neighbor : graph[current]) {
        if (visited.find(neighbor) == visited.end()) {
            parent[neighbor] = current;
            if (dfs(neighbor, destination, visited, parent))
                return true;
        }
    }
    return false;
}
```

TIME COMPLEXITY: O(V + E)
SPACE COMPLEXITY: O(V) for recursion stack

CHARACTERISTICS:
- Explores one branch completely before backtracking
- May find longer paths than necessary
- Useful for graph connectivity testing

EXAMPLE OUTPUT:
DEL → BOM → BLR → MAA → CCU (May take longer route)


ALGORITHM COMPARISON TABLE:
================================================================================
Algorithm    | Time Complexity | Space | Optimal? | Weighted? | Use Case
================================================================================
Dijkstra     | O((V+E) log V) | O(V)  | Yes      | Yes       | Best for optimal path
A*           | O(E log V)     | O(V)  | Yes*     | Yes       | Faster with heuristic
BFS          | O(V + E)       | O(V)  | Yes**    | No        | Minimum hops
DFS          | O(V + E)       | O(V)  | No       | No        | Find any path
================================================================================
* Optimal if heuristic is admissible (never overestimates)
** Optimal only for unweighted graphs

HAVERSINE DISTANCE CALCULATION (C++):
Used to calculate great-circle distance between two coordinates on Earth.

```cpp
#include <cmath>

double haversineDistance(double lat1, double lon1, 
                        double lat2, double lon2) {
    const double R = 6371.0;  // Earth radius in km
    const double PI = M_PI;
    
    double phi1 = lat1 * PI / 180.0;
    double phi2 = lat2 * PI / 180.0;
    double dphi = (lat2 - lat1) * PI / 180.0;
    double dlambda = (lon2 - lon1) * PI / 180.0;
    
    double a = sin(dphi/2) * sin(dphi/2) + 
               cos(phi1) * cos(phi2) * 
               sin(dlambda/2) * sin(dlambda/2);
    double c = 2 * atan2(sqrt(a), sqrt(1 - a));
    
    return R * c;
}
```

GREAT-CIRCLE INTERPOLATION (SLERP):
Generates smooth curved paths on spherical surface.

Process:
1. Convert lat/lon to 3D Cartesian coordinates
2. Apply spherical linear interpolation
3. Convert back to lat/lon coordinates
4. Generate 150 intermediate points for smooth curve

HAVERSINE FORMULA:
a = sin²(Δφ/2) + cos(φ₁) × cos(φ₂) × sin²(Δλ/2)
c = 2 × atan2(√a, √(1−a))
d = R × c

where:
φ = latitude in radians
λ = longitude in radians
R = Earth's radius (6371 km)

GREAT-CIRCLE INTERPOLATION (SLERP):
p(t) = [sin((1-t)Ω) / sin(Ω)] × p₁ + [sin(tΩ) / sin(Ω)] × p₂

where:
Ω = angle between p₁ and p₂
t = interpolation parameter (0 to 1)
p₁, p₂ = 3D Cartesian coordinates

DIJKSTRA IMPLEMENTATION (C++):
```cpp
#include <queue>
#include <unordered_map>
#include <vector>
#include <string>
#include <limits>

vector<string> dijkstra(const string& source, const string& destination, 
                       const string& metric) {
    unordered_map<string, int> distances;
    unordered_map<string, string> parent;
    
    // Initialize distances
    for (const auto& [code, airport] : airports) {
        distances[code] = numeric_limits<int>::max();
    }
    distances[source] = 0;
    
    // Min-heap: pair<distance, node>
    priority_queue<pair<int, string>, 
                   vector<pair<int, string>>, 
                   greater<pair<int, string>>> pq;
    pq.push({0, source});
    
    while (!pq.empty()) {
        auto [currentDist, current] = pq.top();
        pq.pop();
        
        if (current == destination) break;
        if (currentDist > distances[current]) continue;
        
        for (const auto& edge : graph[current]) {
            int weight = (metric == "distance") ? edge.distance : edge.time;
            int newDist = distances[current] + weight;
            
            if (newDist < distances[edge.destination]) {
                distances[edge.destination] = newDist;
                parent[edge.destination] = current;
                pq.push({newDist, edge.destination});
            }
        }
    }
    
    return reconstructPath(parent, source, destination);
}
```

HAVERSINE IMPLEMENTATION (C++):
```cpp
#include <cmath>

double haversineDistance(double lat1, double lon1, 
                        double lat2, double lon2) {
    const double R = 6371.0;  // Earth radius in km
    const double PI = M_PI;
    
    double phi1 = lat1 * PI / 180.0;
    double phi2 = lat2 * PI / 180.0;
    double dphi = (lat2 - lat1) * PI / 180.0;
    double dlambda = (lon2 - lon1) * PI / 180.0;
    
    double a = sin(dphi/2) * sin(dphi/2) + 
               cos(phi1) * cos(phi2) * 
               sin(dlambda/2) * sin(dlambda/2);
    double c = 2 * atan2(sqrt(a), sqrt(1 - a));
    
    return R * c;
}
```

BFS IMPLEMENTATION (C++):
```cpp
#include <queue>
#include <unordered_set>

vector<string> bfs(const string& source, const string& destination) {
    queue<string> q;
    unordered_set<string> visited;
    unordered_map<string, string> parent;
    
    q.push(source);
    visited.insert(source);
    
    while (!q.empty()) {
        string current = q.front();
        q.pop();
        
        if (current == destination) {
            return reconstructPath(parent, source, destination);
        }
        
        for (const auto& edge : graph[current]) {
            if (visited.find(edge.destination) == visited.end()) {
                visited.insert(edge.destination);
                parent[edge.destination] = current;
                q.push(edge.destination);
            }
        }
    }
    
    return vector<string>(); // No path found
}
```
